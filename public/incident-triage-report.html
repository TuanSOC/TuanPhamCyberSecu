Incident Triage Report Sample

This is a sample SOC Tier 1 handoff format. It is not live customer data. It demonstrates how I document signal, evidence, impact, confidence, and next action.

AlertRepeated authentication failures
SeverityMedium
SourceWazuh / pfSense lab
StatusEscalation-ready

Executive Summary

Multiple failed login attempts were observed against a monitored service from a single external source within a short time window. No confirmed successful login was identified in the sample evidence. Recommend blocking or monitoring the source IP and reviewing account exposure if the pattern repeats.

Observed Evidence

  • Repeated authentication failures for the same account family within a concentrated time window.
  • Source IP generated multiple attempts against the same service endpoint.
  • Firewall telemetry showed repeated connection attempts around the alert window.
  • No immediate successful authentication event was confirmed in the reviewed sample.

Analyst Checks

  1. Validate affected user/service and identify whether the account is expected to be exposed.
  2. Correlate SIEM alert with firewall logs, source IP, destination port, and time window.
  3. Search for successful authentication after repeated failures.
  4. Document confidence, false-positive possibility, and recommended containment.

Recommended Next Actions

  • Temporarily block or rate-limit the suspicious source if attempts continue.
  • Review account lockout policy and MFA status for exposed services.
  • Add dashboard note or detection comment: failed_auth_count + source_ip + time_window.
  • Escalate if a successful login appears after the failed attempts.
